Triton hard value

Triton 的工业价值：不是替代 PyTorch，而是补上自定义 kernel 这层

平常写训练代码，直接用 Python / PyTorch 是正确选择。Triton 的位置不是“每个算子都重写一遍”， 而是当 profiler 说明某个 Python 组合算子在 kernel launch 或 HBM 往返上变成瓶颈时，用它写一个融合 kernel。

核心价值： PyTorch 负责表达模型，CUDA 负责最终机器级控制，Triton 夹在中间： 比 PyTorch 更能控制融合和片上数据流，比 CUDA 更快写、更容易 autotune。

0. 你的疑问：我直接写 Python 不就得了？

对，绝大多数时候就该直接写 Python / PyTorch。你们代码库里的 softmax 就是这样：

PyTorch 写法 模型代码里最自然 
def softmax(x, dim):
    x_max = torch.amax(x, dim=dim, keepdim=True)
    exp_x = torch.exp(x - x_max)
    return exp_x / torch.sum(exp_x, dim=dim, keepdim=True)
什么时候才想 Triton profiler 指向它时 
# 默认：继续用 PyTorch
model = Transformer(...)
loss = cross_entropy(...)

# 只有当 profiler 看到：
# 1) 很多小 kernel launch
# 2) 中间 tensor 反复写 HBM
# 3) 某个 shape 高频出现
# 才考虑把这段换成 Triton kernel

分工很简单：

  • PyTorch：最快写出正确模型，自动求导，生态完整。训练主干应该先用它。
  • Triton：把几个 PyTorch ops 融成一个 GPU kernel，控制数据怎么留在片上，减少 HBM 往返。
  • CUDA：极限控制。需要写 shared memory、warp primitive、同步、模板、编译扩展。

所以 Triton 不是“Python 的替代品”。它是 PyTorch profiling 之后的优化工具。 你不会因为会 Triton 就把所有 softmax 都重写；你会在一个模型反复跑、一个 kernel 占了足够多时间时重写它。

1. 真实负载：row-wise softmax

输入是一个二维 tensor：

$$ X \in \mathbb{R}^{M \times N},\qquad M=4096,\quad N=1024 $$

每一行独立做 softmax：

$$ Y_{i,j} = \frac{\exp(X_{i,j}-m_i)} {\sum_{k=0}^{N-1}\exp(X_{i,k}-m_i)}, \qquad m_i=\max_k X_{i,k} $$

Rows
M = 4096
Cols
N = 1024
One row
4 KB fp32
Whole tensor
16 MB

2. 关键差别：HBM 往返次数

CUDA 朴素拆 kernel

读 X→写 max→ 读 X+max→写 exp→ 读 exp→写 sum→ 读 exp+sum→写 Y

中间的 max、exp、sum 都变成 HBM 上的临时结果。

Triton fused kernel

读一行 X→片上 max→ 片上 exp→片上 sum→ 写一行 Y

一个 program 负责一行。中间值留在片上，HBM 只承担输入和输出。

方案 HBM 读写量（粗略） 要点
朴素 CUDA 多 kernel 约 7× tensor 级访问 读写临时 exp；每个 kernel 之间靠 HBM 传中间结果。
Triton fused softmax 约 2× tensor 级访问 读 X 一次，写 Y 一次；中间归约在片上。

3. 实际工作里怎么选

阶段 你通常做什么 原因
第一版模型 PyTorch 原生 op 代码清楚，自动求导稳，改模型最快。
发现慢 先看 profiler，再试 torch.compile / FlashAttention / xFormers / cuDNN 等现成 kernel 已有库 kernel 通常维护得更好，也覆盖很多常见 shape。
现成 kernel 卡住 写 Triton 你的 shape、mask、布局、融合方式很具体；Triton 能快速写一个专用 fused kernel。
极限榨干 CUDA / CUTLASS / 手写汇编级优化 需要完全控制 warp、shared memory、pipeline、指令选择。
这就是为什么工业界用 Triton： 它常常不是训练代码作者每天都要写的东西，而是框架作者、infra 团队、模型性能优化的人用来补齐 “PyTorch 太通用，CUDA 太重” 中间那一层。

4. 左右对照：同一个 softmax

左边不是「高性能 CUDA 最终形态」，而是很多人第一版会写出的拆 kernel 思路。右边是 Triton 的自然写法：一行一个 program。

CUDA 朴素多 kernel 中间结果走 HBM 
// Kernel 1: 每行求 max，写 row_max[M]
__global__ void row_max(X, row_max, M, N) {
  // 一个 block 处理一行；内部做 reduction
  // 结果写回 HBM: row_max[row]
}

// Kernel 2: 读 X 和 row_max，写 exp_tmp[M,N]
__global__ void exp_shifted(X, row_max, exp_tmp, M, N) {
  int idx = blockIdx.x * blockDim.x + threadIdx.x;
  int row = idx / N;
  exp_tmp[idx] = expf(X[idx] - row_max[row]);
  // exp_tmp 是完整临时 tensor，写进 HBM
}

// Kernel 3: 每行求 sum(exp_tmp)，写 row_sum[M]
__global__ void row_sum(exp_tmp, row_sum, M, N) {
  // 再读一遍 exp_tmp；reduction 后写 row_sum[row]
}

// Kernel 4: 归一化，读 exp_tmp + row_sum，写 Y
__global__ void normalize(exp_tmp, row_sum, Y, M, N) {
  int idx = blockIdx.x * blockDim.x + threadIdx.x;
  int row = idx / N;
  Y[idx] = exp_tmp[idx] / row_sum[row];
}
Triton fused row softmax 一行留在片上 
@triton.jit
def softmax_kernel(X, Y, M: tl.constexpr,
                   N: tl.constexpr,
                   BLOCK_N: tl.constexpr):
    # tl = triton.language
    # 一个 program 负责一行
    row = tl.program_id(0)

    # offsets = 这一行的列下标 [0..BLOCK_N)
    cols = tl.arange(0, BLOCK_N)
    mask = cols < N

    # 读一整行到片上向量
    x = tl.load(X + row * N + cols,
                mask=mask, other=-float("inf"))

    # 下面全在片上向量 / 标量归约里完成
    m = tl.max(x, axis=0)
    z = x - m
    num = tl.exp(z)
    den = tl.sum(num, axis=0)
    y = num / den

    # 写最终结果；没有 exp_tmp / row_sum 临时 tensor
    tl.store(Y + row * N + cols, y, mask=mask)

# launch: grid=(M,)
# softmax_kernel[(4096,)](X, Y, M=4096, N=1024, BLOCK_N=1024)

5. 为什么这例子更能说明 Triton

CUDA 当然也能手写融合版

高手可以用 CUDA 写一个 fused softmax：手写 shared memory、warp reduction、block reduction、边界处理、向量化 load/store。 这会比朴素 CUDA 快很多。

代价是源码复杂度快速上升，而且每换一种 shape / dtype / GPU，调参成本也跟着上升。

Triton 的位置

Triton 让你直接写「一行 softmax 的数学数据流」，编译器处理 program 内向量化、寄存器分配、片上归约、mask、生成 GPU kernel。

价值是把 80% 的手写 CUDA 性能工作，变成更短的可读 Python kernel，再配合 autotune 调 tile / warp。

6. 再往工业里看：attention 也是这个模式

Softmax 是 attention 的核心环节。FlashAttention 的大思路也类似： 把一块 $QK^\top$ 分数、softmax 统计量、乘 $V$ 的中间结果尽量留在片上，减少 HBM 临时矩阵往返。

所以 Triton 在工业界常见于：fused softmax、LayerNorm/RMSNorm、FlashAttention、MoE routing、custom matmul、各种「PyTorch 组合算子太多 HBM 往返」的地方。

7. 总结

例子 Triton 的价值 判断
y=2x+1 / GELU 写法更短，按向量写一段 offsets 说明力弱
row-wise softmax 把 max / exp / sum / normalize 融合进一个 kernel，少走 HBM 说明力强
attention / norm / custom matmul 把片上数据流写清楚，让编译器管低层映射，再 autotune 工业常用
