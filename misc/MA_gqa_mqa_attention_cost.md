# GQA/MQA：整段前向 vs 增量解码 —— 算术强度为什么断崖？

对应课上 **「GQA/MQA – Reducing attention head cost」** 连续两页。

公式约定：行内 `$...$`，独立成行 `$$...$$`。  
下面「$O(\cdot)$」表示数量级；需要对照幻灯片原式时会单独写出幻灯片上的括号写法，并标明近似条件。

---

## 核心结论（先读这个）

**Transformer 结构没变，变的是计算模式（使用场景）：**

| | 上一页：整段并行前向 | 下一页：增量串行解码 |
|--|----------------------|----------------------|
| 场景 | 训练前向 / 生成前的 Prefill | 自回归逐 token 生成 |
| 怎么算 | 一次喂入全部 $n$ 个 token，并行算完 | 每次只新增 1 个 token；历史 KV 反复从显存读出 |
| 算力特征 | 数据复用高 → 偏 **计算密集** | 复用差 → 偏 **访存密集** |
| 强度口气 | 高（GPU 吃得饱） | 低（强度暴跌） |

生成满 $n$ 个 token 时：

- **总计算量**（整段一次算完 vs 增量 $n$ 步累加）在数量级上 **几乎相同**：$O(bnd^{2}+bn^{2}d)$  
- **总访存量**在增量路径上会多出「KV 被反复读」这一项，相对「整段只读一轮激活/KV」大约多一个 **$n$ 因子**  
- 于是算术强度相对整段路径大约 **除以 $n$** —— 这就是强度断崖的主因

后文把这句话拆成可核对的账，并在每个对比里同时给出「数字小 / 数字大」两边的结果。

## 0. 符号

| 符号 | 含义 |
|------|------|
| $b$ | batch size |
| $n$ | 序列长度（上下文里已有 / 要生成到的长度） |
| $d$ | hidden size（`d_model`） |
| $h$ | attention head 数 |
| $k$ | 每头宽度，$k=d/h$（故 $hk=d$） |

一层、一次注意力看到的激活粗记：

$$
X \in \mathbb{R}^{b\times n\times d}
$$

K/V 在「所有 head 拼起来」后，每层缓存的总量级是 $O(bnd)$ 个数（MHA 下 K 与 V 各一份，常数 2 吃进 $O$）。

---

## 一、原来的场景：整段并行前向（训练 / Prefill）

### 1.1 计算模式

一次性输入完整 $n$ 个 token，**并行**算出所有位置的注意力输出。

- 训练时的标准前向是这种  
- 生成时的 **Prefill**（先把 prompt 整段吃进去）也是这种

### 1.2 核心优势：访存复用高

- 投影权重 $W_Q,W_K,W_V,W_O$：**读 1 次**，给这 $n$ 个 token 共用  
- K、V：**算 1 次** 写进显存；各位置的 Q 在同一次前向里复用这份 K/V  
- 不会出现「生成第 $t$ 步又把前 $t$ 个 token 的 KV 全部再读一遍」那种累加

### 1.3 量级粗算（幻灯片前提常取 $n<d$）

**总计算量**

| 部分 | 阶 | 说明 |
|------|-----|------|
| Q/K/V/O 线性投影 | $O(bnd^{2})$ | 每份投影 $\sim bnd^{2}$，常数进 $O$ |
| $QK^{\top}$ 与乘 $V$ | $O(bhn^{2}k)=O(bn^{2}d)$ | 每头 $n\times n$ 与头宽 $k$ |

当 $n<d$ 时：

$$
\frac{bn^{2}d}{bnd^{2}}=\frac{n}{d}<1
$$

故 **算术主导项是投影** $O(bnd^{2})$。幻灯片写成 $(bnd^{2})$。

**总访存量（幻灯片上一页）**

$$
(bnd)+(bhn^{2})+(d^{2})
$$

| 项 | 含义 |
|----|------|
| $bnd$ | 读激活 $X$（同阶写输出） |
| $bhn^{2}$ | 物化 $b\cdot h$ 张 $n\times n$ 注意力表（softmax 路径） |
| $d^{2}$ | 投影权重（多份仍是 $\Theta(d^{2})$） |

**算术强度（精确用幻灯片两项相除）**

$$
\frac{bnd^{2}}{bnd+bhn^{2}+d^{2}}
=
\left(\frac{1}{d}+\frac{hn}{d^{2}}+\frac{1}{bn}\right)^{-1}
=
\left(\frac{1}{d}+\frac{n}{kd}+\frac{1}{bn}\right)^{-1}
$$

在课上进一步收到

$$
O\left(\left(\frac{1}{k}+\frac{1}{bn}\right)^{-1}\right)
$$

（丢掉较小的 $1/d$，并把 $\frac{n}{kd}$ 与 $\frac{1}{k}$ 作同阶讨论。）

**若再做更粗的「只留激活访存」直觉**（帮助理解「复用高 → 强度高」，**不是**幻灯片唯一合法式）：

$$
\frac{bnd^{2}}{bnd}=d=O(d)
$$

这与「强度很高、能喂饱 GPU」的口头结论一致；完整分母还含 $bhn^{2}$、$d^{2}$，数值会低一些，但在 $n$ 不太大时仍可保持较高强度。

### 1.4 小例子 vs 大例子（整段强度，两边都算）

取 $b=4$，$d=1024$，$h=16$（故 $k=64$）。用幻灯片上一页完整式  
$\mathrm{AI}=bnd^{2}/(bnd+bhn^{2}+d^{2})$。

| | 短序列 | 长序列 |
|--|--------|--------|
| $n$ | $256$ | $4096$ |
| 算术 $bnd^{2}$ | $4\cdot256\cdot1024^{2}\approx1.07\times10^{9}$ | $4\cdot4096\cdot1024^{2}\approx1.72\times10^{10}$ |
| 访存 $bnd$ | $4\cdot256\cdot1024\approx1.05\times10^{6}$ | $4\cdot4096\cdot1024\approx1.68\times10^{7}$ |
| 访存 $bhn^{2}$ | $4\cdot16\cdot256^{2}\approx4.19\times10^{6}$ | $4\cdot16\cdot4096^{2}\approx1.07\times10^{9}$ |
| 访存 $d^{2}$ | $1.05\times10^{6}$ | $1.05\times10^{6}$ |
| 总访存 | $\approx6.3\times10^{6}$ | $\approx1.09\times10^{9}$ |
| 强度 AI | $\approx\mathbf{170}$ | $\approx\mathbf{16}$ |

两边都算完了：整段路径下 $n$ 变长，**softmax 表 $bhn^{2}$ 会变沉**，强度从约 $170$ 降到约 $16$，但仍远高于后面增量账里「长上下文」那种个位数以下。

粗近似 $bnd^{2}/bnd=d=1024$ 两边相同——它 **看不见** $n$ 变长带来的表访存；所以正式对比仍以带 $bhn^{2}$ 的式子为准。

## 二、现在的场景：增量串行解码（自回归 / Incremental）

### 2.1 计算模式

已缓存前缀的 K/V。每一步 **只新增 1 个 token**：

1. 只对这 1 个 token 做 Q/K/V 投影（新 K/V 追加进 cache）  
2. 用新 Q 对 **cache 里已有全部历史** 做注意力  
3. 输出投影，吐出下一个 token 分布  

要得到长度 $n$ 的续写，这种单步要串行做约 $n$ 次（此处用 $t=1,\ldots,n$ 记步数，便于求和）。

### 2.2 核心问题：历史 KV 被反复读

| 步 $t$ | 这一步大约要读多长的 KV |
|--------|---------------------------|
| $t=1$ | 长度 $\sim1$ |
| $t=2$ | 长度 $\sim2$ |
| $\vdots$ | $\vdots$ |
| $t=n$ | 长度 $\sim n$ |

元素个数（一层、K 与 V 合计吃进常数）每步 $\sim b\,t\,d$，**$n$ 步累加**：

$$
\sum_{t=1}^{n} b\,t\,d
=
bd\cdot\frac{n(n+1)}{2}
=
O(bn^{2}d)
$$

相对「整段前向里 K/V 或激活只按 $O(bnd)$ 过一轮」，这里多了一个求和出来的 **$\Theta(n)$ 倍反复读取**。  
这不是架构改了，是 **使用方式强迫你把同一段历史读很多遍**。

### 2.3 总计算量：和整段几乎同一数量级（可核对）

对步 $t$（cache 长度 $\approx t$）粗计一层：

| 部分 | 单步阶 | $t=1..n$ 求和 |
|------|--------|----------------|
| 新 token 的 Q/K/V/O 投影 | $O(bd^{2})$ | $O(bnd^{2})$ |
| 新 Q 对长度 $t$ 的 K/V | $O(btd)$ | $\sum_t O(btd)=O(bn^{2}d)$ |

故增量路径总计算：

$$
O(bnd^{2}+bn^{2}d)
$$

整段并行前向也是投影 $O(bnd^{2})$ 加注意力 $O(bn^{2}d)$，**总量同阶**。  
幻灯片增量页仍把算术写成 $(bnd^{2})$，是在 $n<d$ 或只强调投影主导时的简写；完整比较应用 $O(bnd^{2}+bn^{2}d)$。

### 2.4 总访存量：主导项变成反复读 KV

| 部分 | 阶 | 说明 |
|------|-----|------|
| 反复读 KV cache | $O(bn^{2}d)$ | 上一小节 |
| 逐步加载 K/V 投影权重 | $O(nd^{2})$ | 每步读 $W_K,W_V\sim d^{2}$，约 $n$ 步 → $nd^{2}$（权重不随 $b$ 复制时，幻灯片写作 $nd^{2}$） |

幻灯片增量页：

$$
\text{Total memory accesses}=(bn^{2}d+nd^{2})
$$

并在 $nd^{2}$ 上标注 **K, V projection**。

### 2.5 算术强度（对照幻灯片）

用幻灯片原式（算术取 $(bnd^{2})$）：

$$
\frac{bnd^{2}}{bn^{2}d+nd^{2}}
=
\left(\frac{n}{d}+\frac{1}{b}\right)^{-1}
=
O\left(\left(\frac{n}{d}+\frac{1}{b}\right)^{-1}\right)
$$

推导：分子分母除以 $bnd^{2}$ 得 $\frac{n}{d}$ 与 $\frac{1}{b}$。

**与「强度约除以 $n$」对齐的粗读**（把分母近似成主导项 $bn^{2}d$）：

$$
\frac{bnd^{2}}{bn^{2}d}=\frac{d}{n}=O\!\left(\frac{d}{n}\right)
$$

相对整段粗强度 $O(d)$，正好 **多除一个 $n$**。  
更完整时分母还有 $nd^{2}$，强度是 $(n/d+1/b)^{-1}$，结论同一方向：$n$ 升或 $b$ 极小都会砸强度。

### 2.6 小例子 vs 大例子（增量强度，两边都算）

固定 $b=4$，$d=1024$。强度用幻灯片式 $(n/d+1/b)^{-1}$。

| | 短解码 | 长解码 |
|--|--------|--------|
| $n$ | $256$ | $4096$ |
| $n/d$ | $0.25$ | $4$ |
| $1/b$ | $0.25$ | $0.25$ |
| $n/d+1/b$ | $0.5$ | $4.25$ |
| 强度 | **$2.0$** | **$\approx0.235$** |

两边都算完：只把目标长度从 $256$ 拉到 $4096$，增量强度从 $2$ 掉到 $\approx0.24$。

再固定 $n=4096$，$d=1024$，对比 batch：

| | 小 batch | 大 batch |
|--|----------|----------|
| $b$ | $1$ | $32$ |
| $n/d$ | $4$ | $4$ |
| $1/b$ | $1$ | $0.03125$ |
| 强度 | **$0.2$** | **$\approx0.248$** |

两边都算完：大 batch 只把强度从 $0.2$ 抬到 $\approx0.25$，**救不动已经很大的 $n/d=4$**。这就是幻灯片说 large batch 有帮助、但 $n/d$ 难减。

## 三、两页公式差异（本质：访存主导项换人了）

| 场景 | 幻灯片强度 | 核心影响因素 |
|------|------------|--------------|
| 整段前向 | $O\bigl((1/k+1/(bn))^{-1}\bigr)$ | $k$ 大、$bn$ 大 → 更好 |
| 增量解码 | $O\bigl((n/d+1/b)^{-1}\bigr)$ | $n$ 小或 $d$ 大、$b$ 大 → 更好；$n$ 大最伤 |

| | 整段前向 | 增量解码 |
|--|----------|----------|
| 算术（幻灯片写法） | $(bnd^{2})$ | $(bnd^{2})$ |
| 访存主导 | 激活 $bnd$、表 $bhn^{2}$、权重 $d^{2}$ | 反复读 KV 的 $bn^{2}d$，外加 $nd^{2}$（K/V 投影权重逐步加载） |
| 总计算 vs 对方 | 同阶 $O(bnd^{2}+bn^{2}d)$ | 同阶 $O(bnd^{2}+bn^{2}d)$ |
| 总访存 vs 对方 | 无「$\sum_t t$」那种 KV 重复读 | KV 读取多 $\Theta(n)$ 倍量级 |

### 并排数字（同一组超参，两页公式都算满）

取 $b=8$，$n=512$，$d=1024$，$h=16$（$k=64$）。

| | 整段页公式 $(1/k+1/(bn))^{-1}$ | 增量页公式 $(n/d+1/b)^{-1}$ |
|--|----------------------------------|------------------------------|
| 代入 | $1/64+1/(8\cdot512)\approx0.01587$ | $512/1024+1/8=0.625$ |
| 强度 | $\approx\mathbf{63}$ | $\mathbf{1.6}$ |

再取长上下文 $n=8192$，其余不变：

| | 整段页公式 | 增量页公式 |
|--|------------|------------|
| 代入 | $1/64+1/(8\cdot8192)\approx0.01564$ | $8192/1024+1/8=8.125$ |
| 强度 | $\approx\mathbf{64}$ | $\approx\mathbf{0.123}$ |

两边都算完了：**不是算术突然少了 $n$ 倍，是增量访存账换了主导项**，同一套硬件叙事下强度可以差出一个 $n$ 量级。

## 四、GQA / MQA 的解决逻辑（对准「KV 体积 × 反复读」）

问题根因：**KV 缓存大，且增量下被读很多次。**  
$n/d$ 里的 $n$ 是产品目标（长上下文），不能靠「别生成那么长」糊弄；能动手的是 **每个位置存多宽的 K/V**。

| 方案 | K/V 头与 Q 头关系 | 每层 KV 缓存宽度（相对 MHA） |
|------|-------------------|------------------------------|
| MHA | 每个 Q 头一套 K/V | $h$ 组，$hk=d$（最满） |
| GQA | 一组 Q 头共享一套 K/V | $g$ 组，$g<h$（折中） |
| MQA | 全部 Q 头共享一套 K/V | $1$ 组，宽度 $k=d/h$（最省） |

### 直观例子：缓存元素数（两边都写清）

一层、只数 **K**（V 同倍；$b=1$；$n=4096$；$d=1024$；$h=16$ ⇒ $k=64$）。

| | MHA | MQA |
|--|-----|-----|
| 形状粗记 | $h\times n\times k$ | $1\times n\times k$ |
| 元素个数 | $16\cdot4096\cdot64=\mathbf{4{,}194{,}304}$ | $1\cdot4096\cdot64=\mathbf{262{,}144}$ |
| 相对 | $1\times$ | **$1/16$** |

GQA 若 $g=4$ 组 K/V：元素个数 $=4\cdot4096\cdot64=\mathbf{1{,}048{,}576}$，介于 MHA 与 MQA 之间（MHA 的 $1/4$，MQA 的 $4$ 倍）。

增量下每一步读 cache 的流量近似与「cache 元素数」成正比，故 MQA/GQA 直接砍的是 **$bn^{2}d$ 里那道与头复制相关的系数**，从而缓解强度不足。  
它 **不会让 $n$ 变小**，但让「同样长度 $n$」更便宜——这才是标题 *Reducing attention head cost* 的落点。

### 对强度公式的精确说法（避免模糊）

- 幻灯片增量式 $(n/d+1/b)^{-1}$ 是 **标准 MHA 量级账** 下的粗强度；没有把「头共享」写进符号。  
- MQA/GQA 改变的是真实系统里 KV 读写的前导常数（以及实现中的带宽），使 **同样 $n,b,d$ 时有效访存小于朴素 $O(bn^{2}d)$**。  
- 因此：不是从代数上抹掉 $n/d$，而是 **降低 $n/d$ 那一项背后的实际 IO 常数**。

## 五、收束对照表

| 问题 | 准确答案 |
|------|----------|
| 架构变了吗？ | **没有**；变的是整段并行 vs 增量串行 |
| 总计算差很多吗？ | **同阶** $O(bnd^{2}+bn^{2}d)$ |
| 强度为何崩？ | 增量把 KV **反复读** 累成 $O(bn^{2}d)$，相对整段多 $\sim n$ 倍访存，强度 $\sim$ 除以 $n$ |
| 上一页 vs 下一页公式 | 访存主导项从「激活+表+权重」换成「KV 反复读 + K/V 投影权重逐步加载」 |
| GQA/MQA 做什么？ | **缩小 KV 缓存宽度（头间共享）**，降低反复读的体积 |

幻灯片原话对齐：增量强度不好，需要 large $b$、短 $n$ 或大 $d$；**$n/d$ 难减** → 从减少 attention head 带来的 KV 代价（GQA/MQA）绕过去。
