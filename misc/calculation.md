# GPT-2 XL 计算

## Q1：参数量与加载显存

配置：

- vocab_size = 50257
- context_length = 1024
- num_layers = 48
- d_model = 1600
- num_heads = 25
- d_ff = 4288

约定：无 bias；RoPE 无训练参数；embedding 与 LM head 不共享权重。

### 交付句（1–2 句）

该配置下一共有 **1,640,452,800** 个可训练参数；若每个参数用 FP32（4 bytes）表示，仅加载权重约需 **6,561,811,200 bytes ≈ 6.11 GiB（约 6.56 GB）**。

### 分项计算

#### 1. Token embedding

```
V * d_model = 50257 * 1600 = 80,411,200
```

#### 2. 每一层 Transformer block

Attention（4 个 d_model×d_model 矩阵：Q、K、V、O）

```
4 * d_model^2 = 4 * 1600 * 1600 = 10,240,000
```

等价看法：25 头 × 3（Q/K/V），每头 `1600 × 64`，再加 `W_O: 1600 × 1600`，总数相同。

FFN / SwiGLU（W1、W2、W3）

```
3 * d_model * d_ff = 3 * 1600 * 4288 = 20,582,400
```

两套 RMSNorm 的缩放向量 γ

```
2 * d_model = 2 * 1600 = 3,200
```

单层合计

```
10,240,000 + 20,582,400 + 3,200 = 30,825,600
```

48 层

```
48 * 30,825,600 = 1,479,628,800
```

#### 3. Final RMSNorm + LM head

```
ln_final:  d_model              = 1,600
lm_head:   V * d_model          = 50257 * 1600 = 80,411,200
```

### 总计

| 部分 | 参数量 |
|------|--------:|
| token embedding | 80,411,200 |
| 48 × Transformer block | 1,479,628,800 |
| ln_final | 1,600 |
| lm_head | 80,411,200 |
| **合计** | **1,640,452,800** |

FP32 加载内存：

```
1,640,452,800 * 4 = 6,561,811,200 bytes
                 ≈ 6.11 GiB
                 ≈ 6.56 GB
```

核对：本仓库 `TransformerLM` 按该配置实例化后，

```
sum(p.numel() for p in model.parameters()) == 1_640_452_800
```

## Q2：前向矩阵乘 FLOPs

配置同上。记 `L = context_length`（本题 `L = 1024`），`d = d_model`，`V = vocab_size`，`N = num_layers`。

### 什么叫「矩阵乘」？FLOP 怎么数？

**点积（dot product）** = 矩阵乘的一个输出格。长度为 `n` 的两向量点积，按 **`2n` FLOPs** 计（`n` 次乘法 + `n-1` 次加法，惯例合并为 `2n`）。

**矩阵乘** `(m × k) @ (k × n)`：`FLOPs = m × n × 2k`。

**逐元素乘（element-wise）** 不求和，**不算矩阵乘**。

| 运算 | 是不是矩阵乘 |
|------|-------------|
| `x @ W^T`、Q/K/V/O 投影 | ✅ |
| `Q @ K^T`、`Attn @ V` | ✅ |
| RMSNorm、SwiGLU 门控 `⊙`、Embedding 查表 | ❌ |

### 前向里有哪些矩阵乘？

**不计入**：embedding 查表、RMSNorm、残差、softmax、RoPE、SwiGLU 逐元素门控。

#### 每层 Attention

| 矩阵乘 | FLOPs |
|--------|------:|
| `W_Q`, `W_K`, `W_V`, `W_O` 各 `(L,d)@(d,d)` | 各 `2Ld²`，合计 `8Ld²` |
| `Q @ K^T`（所有头） | `2L²d` |
| `Attn @ V`（所有头） | `2L²d` |

#### 每层 FFN / SwiGLU

| 矩阵乘 | FLOPs |
|--------|------:|
| `W1`, `W3`：`(L,d)@(d,d_ff)` | 各 `2Ld·d_ff` |
| `W2`：`(L,d_ff)@(d_ff,d)` | `2Ld_ff·d` |

三层合计 **`6Ld·d_ff`**。`SiLU(W1x) ⊙ W3x` 是矩阵乘之后的逐元素运算。

#### 全模型

```
Total = N · (8Ld² + 4L²d + 6Ld·d_ff) + 2LdV
```

最后一项是 LM head `(L,d)@(d,V)`。

### 代入 `L = 1024`

每层：

```
8Ld²     = 20,971,520,000
4L²d     =  6,710,886,400
6Ld·d_ff = 42,253,056,000
─────────────────────────
单层     = 69,835,161,600
```

```
48 × 69,835,161,600 + 2 × 1024 × 1600 × 50257
= 3,516,769,894,400  ≈ 3.52 × 10¹² FLOPs
```

### 交付句

对长度为 `L` 的输入，所有矩阵乘共需 **`N (8Ld² + 4L²d + 6Ld·d_ff) + 2LdV`** 次浮点运算；`L = 1024` 时约为 **3.52 × 10¹² FLOPs**。

## Q3：哪些部分最耗 FLOPs？

`L = 1024` 时，总 FLOPs = **3,516,769,894,400**。

| 部分 | FLOPs | 占全模型 |
|------|------:|--------:|
| **MHA**（Q/K/V/O 投影 + `Q@K^T` + `Attn@V`） | 1,328,755,507,200 | **37.78%** |
| **FFN**（SwiGLU 的 W1/W2/W3） | 2,023,332,249,600 | **57.53%** |
| **LM head** | 164,682,137,600 | **4.68%** |

MHA 内部分解：

| 子项 | FLOPs | 占 MHA | 占全模型 |
|------|------:|-------:|--------:|
| Q/K/V/O 投影（`8Ld²`） | 1,006,632,960,000 | 75.76% | 28.62% |
| `Q@K^T` + `Attn@V`（`4L²d`） | 322,122,547,200 | 24.24% | 9.16% |

### 交付句（1–2 句）

在 `L = 1024` 时，前向矩阵乘 FLOPs 中 **FFN 占 57.53%**，**MHA 占 37.78%**（其中 Q/K/V/O 投影 28.62%，attention 核心 `Q@K^T` 与 `Attn@V` 合计 9.16%），**LM head 占 4.68%**；FFN 是最大头，MHA 次之，LM head 最小。

## Q4：GPT-2 small / medium / large / XL 的 FLOPs 占比

约定：`L = 1024`，`V = 50257`，`d_ff = round_up_64(⌊8/3 · d⌋)`。公式：

```
Total = N · (8Ld² + 4L²d + 6Ld·d_ff) + 2LdV
```

顺序：small → medium → large → XL（由小到大）。XL 即 Q1–Q3 的 GPT-2 XL。

### 配置（四模型对齐）

| 模型 | N（层数） | d（d_model） | heads | d_ff |
|------|----------:|-------------:|------:|-----:|
| GPT-2 small | 12 | 768 | 12 | 2048 |
| GPT-2 medium | 24 | 1024 | 16 | 2752 |
| GPT-2 large | 36 | 1280 | 20 | 3456 |
| GPT-2 XL | 48 | 1600 | 25 | 4288 |

### 组件占比（占该模型前向矩阵乘总 FLOPs；四模型同一表）

| 模型 | N | d | heads | d_ff | MHA | FFN | LM head | 总 FLOPs |
|------|--:|--:|------:|-----:|----:|----:|--------:|---------:|
| small | 12 | 768 | 12 | 2048 | 33.13% | 39.76% | 27.10% | 2.92×10¹¹ |
| medium | 24 | 1024 | 16 | 2752 | 37.25% | 50.05% | 12.70% | 8.30×10¹¹ |
| large | 36 | 1280 | 20 | 3456 | 37.86% | 54.76% | 7.37% | 1.79×10¹² |
| XL | 48 | 1600 | 25 | 4288 | 37.78% | 57.53% | 4.68% | 3.52×10¹² |

MHA 再拆开（占全模型；四模型同一表）：

| 模型 | N | d | Q/K/V/O 投影 | `Q@K^T` + `Attn@V` | MHA 合计 |
|------|--:|--:|-------------:|-------------------:|---------:|
| small | 12 | 768 | 19.88% | 13.25% | 33.13% |
| medium | 24 | 1024 | 24.83% | 12.42% | 37.25% |
| large | 36 | 1280 | 27.04% | 10.82% | 37.86% |
| XL | 48 | 1600 | 28.62% | 9.16% | 37.78% |

### 交付句（1–2 句）

随着模型变大（small → medium → large → XL），**FFN 占比持续上升**（39.76% → 50.05% → 54.76% → 57.53%），**LM head 占比明显下降**（27.10% → 12.70% → 7.37% → 4.68%）；**MHA 占比先升后大致稳定在 ~38%**，其中 Q/K/V/O 投影占比上升，attention 核心（`4L²d`）占比略降。

## Q5：GPT-2 XL，把 context length 提到 16,384

配置仍为 XL：`N=48`，`d=1600`，`heads=25`，`d_ff=4288`，`V=50257`。对比 `L=1024` vs `L=16384`：

| L | 总 FLOPs | MHA | FFN | LM head | 其中投影 | 其中 `Q@K^T`+`Attn@V` |
|--:|---------:|----:|----:|--------:|---------:|----------------------:|
| 1024 | 3.52×10¹² | 37.78% | 57.53% | 4.68% | 28.62% | 9.16% |
| 16384 | 1.34×10¹⁴ | 73.79% | 24.24% | 1.97% | 12.06% | 61.73% |

（`L` 放大 16 倍：线性项约 ×16，`L²` 的 attention 核心约 ×256；总 FLOPs 约变为原来的 **38.0 倍**。）

### 交付句（1–2 句）

把 GPT-2 XL 的 context length 从 1024 增到 16,384 后，单次前向总 FLOPs 约增至 **3.52×10¹² → 1.34×10¹⁴（约 38 倍）**；相对占比上，**MHA 升至约 73.8%**（几乎全由 `Q@K^T` / `Attn@V` 的 `L²` 项推动，该子项从 9.2% 升到 61.7%），而 **FFN、投影与 LM head 的占比都下降**。
